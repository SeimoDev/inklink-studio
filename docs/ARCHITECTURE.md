# InkLink Web-only 架构

## 边界

InkLink 由两个可独立构建、通过 USB 协议协作的部分组成：

1. `web/` 负责所有用户交互、排版、定时任务、固件备份和刷写。
2. `firmware/` 负责设备配置、传感器采集、帧校验、坐标旋转和 UC8251D 刷新。

项目没有常驻后端、云端账户、命令行用户入口或设备端 Web 服务。

## 运行数据流

```text
浏览器编辑器
  → 296×152 Canvas
  → 1-bit / 5,624 bytes / CRC32
  → Web Serial USB
  → ESP32-C3 协议层
  → 152×296 原生缓冲区
  → UC8251D 全刷、局刷或无变化跳过
```

传感器和设备配置沿相反方向通过单行 JSON 返回。网页依据设备保存的间隔读取数据并重新渲染画面，每次以 `auto` 发送完整帧；固件最终判断使用局刷、维护性全刷或跳过。刷新策略和上一帧由设备端掌握，因此断开并重新连接 Web Serial 不会让网页强制全刷；若设备实际重启并丢失帧基线，则安全地全刷一次。

## Web 模块

```text
main.ts
├── core/       数据结构、屏幕参数、当前固件版本与哈希
├── editor/     图片处理、Canvas 图层和帧编码
└── device/     串口、协议、传感器、变化区域与浏览器刷写
```

`main.ts` 是组合层，不承载底层图像或协议算法。可测试的纯逻辑保留在各领域模块中，由 `tests/` 覆盖。

## 固件模块

| 模块 | 职责 |
| --- | --- |
| `app_main.c` | 初始化和命令循环 |
| `protocol.*` | 文本命令、JSON 响应和二进制帧接收 |
| `device_config.*` | NVS 配置和范围校验 |
| `sensors.*` | ADC、温度、VBUS、运行状态 |
| `radios.*` | Wi-Fi/BLE 控制器启停 |
| `epd_uc8251.*` | 屏幕时序、坐标旋转、差异窗口和刷新 |
| `crc32.*` | 帧完整性校验 |

## 固件分发

当前合并固件只保留一份在 `web/public/firmware/`，作为 Web 应用的静态资源。`release/`、重复 ZIP 和独立刷写脚本不属于源码工程；生产网站由 `web/dist/` 临时构建生成。

原设备整片备份包含 NVS 和潜在凭据，只保存在 Git 忽略的 `backups/` 中，永远不作为网页静态资源发布。
